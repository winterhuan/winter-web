Transformer 读论文速查

读《Attention Is All You Need》时，不要先陷进所有公式。先抓住论文的主张、替换掉的旧机制，以及新架构带来的工程收益。 如果英文原文读起来吃力，先用 Transformer 英文论文伴读 拆句，再回到原文。

前置词汇

  • sequence transduction：把一个序列转换成另一个序列，例如机器翻译、语音识别、摘要生成。
  • recurrence：循环结构，典型代表是 RNN/LSTM/GRU；它按顺序一步步读 token。
  • convolution：卷积结构，典型代表是 CNN；它用局部窗口扫描序列。

一页读法

  1. 问题：机器翻译这类 sequence transduction 任务，以前主要靠 RNN/CNN 加 attention。
  2. 主张：Transformer 只用 attention，去掉 recurrence 和 convolution。
  3. 关键机制：self-attention 让序列中每个 token 直接读取其他 token 的信息。
  4. 工程收益：路径更短、并行性更好，更适合大规模训练。
  5. 代价：需要 positional encoding 表达顺序，attention 的计算量随序列长度增长较快。

优先读这些章节

  • Abstract: 论文的一句话贡献。
  • Introduction: 为什么要摆脱 recurrence。
  • 3 Model Architecture: encoder、decoder、attention、feed-forward、positional encoding。
  • 3.2 Attention: scaled dot-product attention 和 multi-head attention。
  • 4 Why Self-Attention: 为什么 self-attention 值得替代旧结构。

博客写作骨架

  1. 旧世界：RNN 逐步读序列，长距离依赖和并行训练都不舒服。
  2. 新问题：能不能让每个 token 一次性“看见”相关 token？
  3. 核心答案：self-attention 用 query/key/value 计算信息路由。
  4. 架构组合：attention 负责混合上下文，feed-forward 负责逐位置变换，positional encoding 补顺序。
  5. 产品含义：LLM 的能力来自大规模模式建模，不等于可靠规划、长期记忆或真实工具执行。
主要来源： Vaswani et al., Attention Is All You Need, Jay Alammar, The Illustrated Transformer, Stanford CS224N Lecture 8
