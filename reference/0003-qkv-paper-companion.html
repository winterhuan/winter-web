Q/K/V 论文伴读

这一页帮你读《Attention Is All You Need》第 3.2 节。目标是先读懂 Q/K/V 在公式里的角色，再进入 scaled dot-product attention。

配合阅读： 《Attention Is All You Need》 的第 3.2 节 Attention，尤其是 3.2.1 Scaled Dot-Product Attention。 对应课程： Q/K/V：self-attention 的最小直觉。

先抓主句

原文会说：attention function 可以被描述成把一个 query 和一组 key-value pairs 映射到一个 output。不要先卡在 function、mapping 这些词上，主干是：

attention 接收：一个 query + 多个 key/value 对；attention 输出：一个加权合成后的结果。

术语块拆解

query 当前 token 发出的“我要找什么”。读公式时把它看成检索请求。
keys 其他 token 提供的“可匹配索引”。query 会和每个 key 算匹配分数。
values 真正被读取并混合的信息。分数高的 value 权重更大。

英文句子怎么转中文

当你看到 mapping a query and a set of key-value pairs to an output，不要翻成僵硬的“将一个查询和一组键值对映射到输出”。更适合学习和写博客的说法是：

attention 做的是一次软检索：拿 query 去匹配每个 key，再按匹配强度加权读取对应的 value，最后得到一个汇总后的向量。

公式前的最小词表

  • function：函数；这里指 attention 这个计算过程。
  • mapping：映射；输入一些向量，输出一个新向量。
  • compatible：匹配程度高；query 和 key 越 compatible，权重越大。
  • weighted sum：加权求和；不是只选一个 value，而是按权重混合多个 value。
  • scaled：缩放；后面公式里会用维度的平方根做缩放，避免分数过大。

博客化表达

如果要给工程读者解释 Q/K/V，可以这样写：self-attention 像一次可微的检索。每个 token 先提出自己的 query，再和上下文里所有 token 的 key 计算相关性，最后按相关性混合它们的 value。这样，每个 token 都能动态决定该从上下文中读取什么。
