Lesson 0001

Transformer 解决的核心问题

这一课只解决一个问题：为什么《Attention Is All You Need》会把“去掉 recurrence 和 convolution”当成核心贡献？

本课主源： 《Attention Is All You Need》。 辅助阅读： The Illustrated Transformer 和 Stanford CS224N Lecture 8。

读论文前的三个词

这篇论文的开头会连续出现几个旧 NLP 架构词。先把它们压成工程直觉，后面读 Abstract 和 Introduction 会轻很多。

sequence transduction 把一个序列转换成另一个序列。比如英文句子转中文句子、语音转文字、文章转摘要。
recurrence 循环结构。RNN/LSTM/GRU 这类模型按顺序一步步读 token，后一步依赖前一步的状态。
convolution 卷积结构。CNN 用固定大小的局部窗口扫描序列，先抓局部模式，再靠堆层传播远距离信息。

所以论文开头那句话可以翻译成：过去做“输入序列到输出序列”的主流模型，主要靠一步步读的 RNN，或局部窗口扫描的 CNN，再加上 attention。

先抓住论文的对手

原论文的第一句话就给了上下文：当时主流 sequence transduction 模型基于复杂的 recurrent 或 convolutional neural networks，通常再接 attention。Transformer 的主张不是“attention 很有用”这么宽泛，而是更激进：只用 attention，完全去掉 recurrence 和 convolution。

旧路径 RNN/CNN 负责混合序列信息，attention 作为补充。
问题 长距离依赖路径长；训练并行性受限；结构复杂。
新答案 self-attention 直接让 token 彼此读取信息。

一句话模型

你可以先把 self-attention 理解成一个信息路由机制：每个 token 发出一个 query，拿它去匹配其他 token 的 key，然后按匹配强度读取对应的 value。

这句话足够你读论文的前半部分：query 是“我在找什么”，key 是“我提供什么索引”，value 是“我真正贡献什么信息”。

为什么这对 Agent 产品也重要

Transformer 擅长从上下文中路由和组合信息，但这不等于它天然拥有可靠目标管理、长期记忆、工具执行或产品级纠错。做 Agent 时，模型是推理与语言接口；工具、状态、评估和边界条件仍然要由系统设计承担。
